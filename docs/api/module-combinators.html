(static) compose(f, g)

Source:

Composes two functions.

This is a direct implementation of the B combinator, composing two single-argument functions, except that it is optionally curried. To compose more than two functions at once, either compose can be chained or composeAll can be used.

Composition is perhaps the most basic operation of functional programming (after function application). It is widely used in many, many languages. Its Haskell equivalent is ., its APL equivalent is âˆ˜ (which is the mathematical symbol for composition), and stack-based languages make composition so basic as to do it by literally putting two functions next to each other in the code.

Composing traditionally happens in right-to-left order (function g will be applied to value x, and then function f is applied to that result). For left-to-right order, use pipe instead.

// Turns the digits in an array into strings and then concatenates them
const stringify = compose(reduce(add), map(string))
const result = stringify([3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5])
console.log(result)   // "31415926535"

Name Type Description
f function

The second function executed, applied to the result of g's application of x. The result of this function will be compose's return value.
g function

The first function executed, applied to x. Its result becomes the argument to f.
Returns:

Function f's result when applied to the result of g being applied to x.

(static) composeAll(…fns) → {function}

Combines two or more single-parameter functions into one single-parameter function that passes the results from its contained functions to the next and returns the result of the last contained function.

This is an implementation of the B combinator except that it can compose more than two functions. It is the equivalent of chaining compose an arbitrary number of times.

The order of the functions is reversed, as is traditional in functional programming. If you need the order to not be reversed, use pipeAll.

(static) pipe(f, g)

Source:
  • combinators/pipe.js, line 33

Composes two functions in left-to-right order.

This is the same as compose except for the left-to-right order replacing compose's right-to-left order. That makes it an implementation of the **Q combinator, or alternately, the same as swap(compose).

This is then similar to Clojure's ->> threading macro.

// Same as example in `compose`, for contrast
// Turns the digits in an array into strings and then concatenates them
const stringify = pipe(map(string), reduce(add))
const result = stringify([3, 1, 4, 1, 5, 9, 2, 6, 5, 3, 5])
console.log(result)   // "31415926535"

(static) pipeAll(…fns) → {function}

Source:

Combines two or more single-parameter functions into one single-parameter function that passes the results from its contained functions to the next and returns the result of the last contained function.

This is an implementation of the B combinator except that the order of the composed functions is reversed and that it can All more than two functions. (In the case of three or more, it's the same as using the B combinator multiple times.)

The order of the functions is not reversed. The first function listed is the first applied. This is contrary to the order most often used in functional programming (and opposite of the order in composeAll), but there are times when the piped order makes more sense than the composed order.

(static) rotl(fn, x, y, z) → {function}

Source:
  • combinators/rotl.js, line 42

Rotates a three-parameter function's arguments so that the first parameter is applied last.

To rotate the arguments in the other direction, use rotr.

This is an implementation of the R* combinator. It takes its name from Factor's rot function; Factor's function to rotate in the opposite direction is -rot and we can't use a - in a function name in JavaScript, so we use rotl and rotr instead.

rotl makes the first argument given into the last argument applied. This is very useful for functions that take their data as their first argument; it can turn them into functions which take their data as their last argument, which is in many cases easier to compose.

const sub = (x, y, z) => x - y - z

// These are all equivalent and all print 6 to the console
console.log(rotl(sub, 1, 5, -2)) // evaluate directly
console.log(rotl(sub, 1, 5)(-2)) // produce 1-argument function
console.log(rotl(sub, 1)(5, -2)) // produce 2-argument function
console.log(rotl(sub)(1, 5, -2)) // produce 3-argument function

(static) rotr(fn, x, y, z) → {function}

Source:
  • combinators/rotr.js, line 43

Rotates a three-parameter function's arguments so that the last parameter is applied first.

To rotate the arguments in the other direction, use rotl.

This is an implementation of BR*R*, or applying R* twice to result in the opposite rotation from R* itself. It takes its name from Factor's rot function; Factor's actual function to rotate arguments in this direction is -rot and we can't use a - in a function name in JavaScript, so we use rotl and rotr instead.

rotr turns the last argument given into the first argument applied. This is in particular useful in conjunction with the cond operator, where often the true and false (second and third) arguments are partially applied and you want to leave the conditional (the first argument) as the last argument.

const sub = (x, y, z) => x - y - z

// These are all equivalent and all print -8 to the console
console.log(rotr(sub, 1, 5, -2)) // evaluate directly
console.log(rotr(sub, 1, 5)(-2)) // produce 1-argument function
console.log(rotr(sub, 1)(5, -2)) // produce 2-argument function
console.log(rotr(sub)(1, 5, -2)) // produce 3-argument function

(static) swap(fn, x, y) → {function}

Source:
  • combinators/swap.js, line 39

Flips the order of the arguments on a two-argument function.

This is an implementation of the C combinator that dispenses with the requirement that its function argument is a curried function. swap will work with both. The result, either way, is a curried function with the argument order reversed.

swap itself comes pre-curried so it's easy to generate a unary function out of a binary function with its second argument applied instead of its first.

This function is called flip in Haskell, but swap is the name of the same function in Factor. Since many of the other argument-shuffing functions also come from Factor, I have retained that here.

const sub = (x, y) => x - y

// These are all equivalent and all print 4 to the console
console.log(swap(sub, 1, 5)) // evaluate directly
console.log(swap(sub, 1)(5)) // produce 1-argument function
console.log(swap(sub)(1, 5)) // produce 2-argument function
